## v1.11.1 - 2026-05-12
### Fixed
* ClientVersion now uses debug.ReadBuildInfo() to reflect the version a consumer actually imported (e.g. "v1.10.0"), rather than the hardcoded "development" string. Falls back to "development" in local and test builds where build info is unavailable.

## v1.11.0 - 2026-05-12
### Added
* AuraAPIClient.Close() drains idle HTTP connections held by the underlying transport. Call via defer after NewClient to avoid leaking file descriptors.
* Three new functional options: WithHTTPClient injects a custom *http.Client transport; WithUserAgent overrides the default User-Agent header; WithDefaultHeaders sets additional headers on every API request (Authorization, Content-Type, and User-Agent are silently ignored to prevent credential leakage).
### Changed
* Renamed CmekService to CMEKService, client.Cmek to client.CMEK, GetCmeksResponse to GetCMEKsResponse, and GetCmeksData to GetCMEKsData to follow Go acronym naming conventions. This is a breaking change.
* Renamed QueryMetrics.QueriesPerSecond to QueryExecutionTotal to accurately reflect that the value is a cumulative counter (neo4j_db_query_execution_success_total), not a per-second rate. JSON tag updated from `queries_per_second` to `query_execution_total`. This is a breaking change.
### Fixed
* CMEKService.List now sends `?tenant_id=` (not `?tenantID=`) so the Aura API correctly applies the tenant filter
* GetMetricValue now logs at Debug level (not Error) when a metric name is not found. Missing metrics are expected on certain Neo4j plans and versions and are not error conditions.
* OverwriteFromSnapshot now validates that sourceSnapshotID is a valid UUID, consistent with OverwriteFromInstance. Previously only an empty string was rejected.

## v1.10.0 - 2026-04-30
### Changed
* GetGDSSessionData.Create (string) renamed to CreatedAt (time.Time) and Expiry (string) renamed to ExpiresAt (time.Time); custom UnmarshalJSON parses RFC3339 timestamps from the API — callers no longer need to parse timestamps manually
* assessHealth now produces three severity tiers: healthy, warning, and critical. CPU > 95%, memory > 95%, connection usage > 95% (when known), and page cache hit rate < 20% all trigger critical. Previous warning thresholds are unchanged. Critical severity is monotonic — it is never downgraded by a lower-severity condition evaluated afterwards.
* GetSnapshotData.Timestamp changed from string to time.Time; custom UnmarshalJSON parses the RFC3339 string returned by the Aura API — callers no longer need to parse timestamps manually

## v1.9.1 - 2026-04-30
### Changed
* GetGDSSessionResponse.Data changed from []GetGDSSessionData to GetGDSSessionData — the GDS Get and Create endpoints return a single session object, not an array; callers accessing result.Data[0] should update to result.Data
### Fixed
* httpclient: configure explicit http.Transport with production-suitable connection pool settings — MaxIdleConnsPerHost raised from Go default of 2 to 20, preventing connection exhaustion under concurrent load to the same host; TLS and continue timeouts also set explicitly
* prometheus: removed hardcoded MaxConnections=100 assumption — connection usage is now derived from the neo4j_dbms_bolt_connections_max_count metric; when the metric is absent (plan-dependent) MaxConnections and UsagePercent stay at 0 and the connection threshold check is skipped
* validateCreateInstanceConfig now wraps the error from ValidateTenantID using %w, preserving the error chain for callers using errors.Is / errors.As

## v1.9.0 - 2026-04-29
### Added
* Added WithInsecureBaseURL option that bypasses the HTTPS enforcement of WithBaseURL — intended for local development and in-process testing (e.g. httptest.Server) only; never use against a real Aura environment
### Changed
* Renamed error_types.go to errors.go — follows Go convention for files holding error type definitions
* Merged client_types.go into client.go — AuraAPIClient struct, config, and functional options are now co-located with NewClient
* Merged {domain}_service.go + {domain}_types.go into single domain files (instances.go, tenants.go, snapshots.go, cmek.go, graphanalytics.go, prometheus.go) — each domain is now self-contained
* Merged test_helpers_test.go and test_helpers_types_test.go into a single mocks_test.go — mock type definitions and their method implementations are now co-located
* Renamed gds_sessions_service.go / gds_sessions_types.go / gds_sessions_test.go to graphanalytics.go / graphanalytics_test.go — name now matches the Aura API path (graph-analytics) and the AuraAPIClient field (GraphAnalytics)
* Renamed internal/httpClient package to internal/httpclient — package name now follows Go's all-lowercase convention; import path updated from internal/httpClient to internal/httpclient across all consumers
* Renamed instance_test.go to instances_test.go for consistency with the new instances.go source file
* Renamed blackbox_test.go to integration_test.go — the name now describes what the tests are (httptest.Server integration tests) rather than how they are structured (external package)
* Simplified internal/api file names: api_service.go → api.go, api_types.go → types.go, api_service_test.go → api_test.go — the _service suffix was redundant given the package is already named api
* Updated .gitignore to remove stale internal/auraClient/coverage.out entry and confirm internal/httpclient/coverage.out is correctly excluded
### Removed
* Deleted all tombstone/no-op files (test_helpers.go, test_helpers_types.go, api_test.go, and the empty _service.go/_types.go stubs) and coverage.out from source control — these files served only as migration markers and are no longer needed
### Fixed
* httpclient: replaced retryablehttp's default CheckRetry policy with a network-only policy that retries solely on connection-level errors — HTTP responses (including 5xx) are now always returned to the caller rather than being treated as retryable failures

## v1.8.2 - 2026-04-23
### Added
* `User-Agent` header now includes the library version (e.g. `aura-go-client/v1.8.0`) for easier identification in API server logs
### Fixed
* `TruncateString` now counts runes instead of bytes, preventing truncation mid-multibyte UTF-8 character
* URL construction now trims stray leading/trailing slashes before joining, preventing double-slash in API endpoint paths
* Added correctly-spelled `StatusRestoring` constant; retained misspelled `StatusRestroying` as a deprecated alias for backwards compatibility
* `gDSSessionService.Get` and `Delete` now return an error for an empty session ID rather than sending a malformed request to the API
* `GetInstanceHealth` no longer stacks two independent timeouts; it calls the internal `doFetchRawMetrics` directly so the context deadline is applied exactly once
* `tenantService.List`, `Get`, and `GetMetrics` now check for a pre-cancelled context at entry, consistent with all other service methods
* Removed redundant HTTP 200-status check in `FetchRawMetrics` — the API layer already converts any non-2xx response to an error before returning
* Mock types moved from `internal/api` and `internal/httpClient` to `internal/testutil` — mocks no longer compile into production builds
* Removed `StatusStopped` constant whose value was identical to `StatusPaused`, making it impossible to distinguish the two states
* Duplicate compiled UUID regexes for tenant and snapshot IDs consolidated into a single shared `uuidRegex`
* Removed unreachable validation guard in `OverwriteFromInstance` — the preceding empty-string check made the inner `if sourceInstanceID != ""` block impossible to reach
* Compile error in `restoreFromSnapshot` example fixed — `nil` now passed correctly to `Snapshots.List` instead of an untyped string constant
* `GetMetricValue` now returns a descriptive error rather than panicking when called with a nil metrics response
* `test_helpers.go` and `test_helpers_types.go` converted to `_test.go` files — test mocks no longer compile into the production binary
* `gDSSessionService.List` and `Get` now log cancelled-context errors consistently with all other service methods
* `cmekService.List` tenant ID validation simplified — fragile length-based switch with incorrect error message replaced with a direct call to `ValidateTenantID`
* `gDSSessionService.Create` and `Estimate` now return a clear error immediately when called with a nil request, rather than marshalling `null` to the API
### Security
* Added `CreateInstanceData.String()` to redact the `Password` field, preventing credentials leaking via logs or fmt.Printf
* `WithBaseURL` now rejects non-HTTPS URLs to prevent OAuth tokens being transmitted in plaintext
